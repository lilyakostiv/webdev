10. Процедури

1. Процедури без параметрів

Є ситуації, коли в різних місцях програми потрібно виконувати один і той самий алгоритм:

max(a,b,c) ---> max(a,b)->d + max(d,c)

Для забезпечення компактності та наглядності програми Паскаль дозволяє виділити будь-який частковий алгоритм із основного тексту програми і записати його тільки один раз, представивши цей частковий алгоритм в якості самостійного програмного об’єкта, що називається процедурою (або підпрограмою).

Процедура вводиться у використання (визначається) за допомогою опису процедури в розділі процедур і функцій програми:<заголовок процедури>;<блок>;

Заголовок процедури починається із службового слова procedure, містить ім’я процедури.

Для активізації процедури (тобто для виконання фрагменту програми, оголошеного процедурою) в потрібному місці програми служить оператор процедури:

<ім'я процедури>;

Різні особливості опису процедур і способи їх використання будемо розглядати на прикладі такої задачі.

Задача: по заданих дійсних значеннях x і y обчислити u=max(x+y,x*y), v=max(0.5,u).

Програму розв'язку цієї задачі без використання процедур можна записати у вигляді:

program MAX1;

var x, y, u, v: real;

begin read (x, y);

if x+y >x*y then u:=x+y else u:=x*y;

if 0.5 > u then v: = 0.5 else v: = u;

writeln (‘u=’, u, ‘v=’, v)

end.

Тут два рази повторюєтья один і той самий алгоритм: знаходження більшого з двох заданих дійсних значень і присвоєння деякій змінній одержаного результату.

Щоб підкреслити подібність цих двох умовних операторів, запишемо цю програму в такому вигляді:

program MAX2;

var x, y, u, v, a, b, s: real;

begin read (x, y);

a: = x + y; b: = x*y;

if a > b then s: = a else s: = b;

u: = s;

a: = 0.5; b: = u;

if a > b then s: = a else s: = b;

v: = s;

writeln (‘u=’, u, ‘v=’, v)

end.

Тут  два умовні оператори співпадають. Щоб не записувати цей оператор два рази, оголосимо його процедурою з ім’ям MAXPR:

program MAX3;

var x, y, u, v, a, b, s: real;

procedure MAXPR;

begin if a > b then s: = a else s: = b end;

begin read (x, y);

a: = x + y; b: = x*y;

MAXPR;

u: = s;

a: = 0.5; b: = u;

MAXPR;

v: = s;

writeln (‘u=’, u, ‘v=’, v)

end.

2. Процедури з параметрами. Параметри-значення

procedure MAXPR (r1, r2: real);

  begin

if r1 > r2 then s: = r1 else s: = r2

  end;

r1, r2 – формальні параметри процедури, бо вони представляють не якісь конкретні значення, а значення “взагалі”. При кожному зверненні до процедури її формальні параметри конкретизуються.

Оскільки формальні параметри мають тип real, то фактичним параметром може бути будь-який арифметичний вираз.

Встановлюється відповідність між формальними і фактичними параметрами.

Тоді програму запишемо так:

program MAXD;

var x, y, u, v, s: real;

procedure MAXPR (r1, r2: real);

  begin if r1 > r2 then s: = r1 else s: = r2 end;

begin read (x, y);

MAXPR (x + y, x*y); u: = s;

MAXPR (0.5,u); v: = s;

writeln (‘u=’, u, ‘     v=’, v)

end.

Виконання процедури MAXPR (x + y, x*y) рівносильне виконанню еквівалентного йому блоку:

var r1, r2: real;

begin r1: = x + y; r2: = x*y;

          if r1 > r2 then s: = r1 else s: = r2

end.

Розглянуті тут формальні параметри процедури називаються параметрами-значеннями, оскільки кожний з них у тілі процедури набуває значення, яке йому задають під час звертання за допомогою відповідного фактичного параметра.

 

3. Процедури з параметрами. Параметри-змінні

Глобальні та локальні ідентифікатори

У попередній процедурі знайдений результат завжди присвоюється змінній s, а потрібно u і v.

Формальний параметр res відрізняється від r1, r2. Це не значення, а змінна.

Щоб відрізнити параметр-змінну від параметра-значення, перед ним в списку формальних параметрів записується службове слово var. Для параметра-змінної фактичним параметром може бути тільки змінна!

Програму запишемо так:

Program Max3;

Var x,y,u,v:real;

Procedure Max(r1,r2:real;var res:real);

Begin

 If r1>r2 then res:=r1 else res:=r2

End;

Begin

 Read(x,y);

 Max(x+y,x*y,u);

 Max(0.5,u,v);

 Writeln(u,v)

End.

 

Принцип локалізації: імена, що вводяться в будь-якій процедурі, мають силу тільки в цій процедурі.

Формальні параметри. Ці змінні існують тільки на час виконання тіла процедури, а після виходу з процедури вони припиняють своє існування.

Глобальні ідентифікатори в тілі процедури мають той самий зміст, що і до моменту входу в процедуру.

Локальні ідентифікатори. Це ідентифікатори, що не є формальними парметрами, але описані в тілі процедури і діють тільки в процедурі.
